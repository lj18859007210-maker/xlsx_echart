## 二、结构确认页 UI 重做（P1）

### 2.1 任务概览卡片
**现状：** 任务信息只用 overview-card 简单罗列，无设计感。
**目标：** 设计成状态指示卡片组：
- 任务编号用大字 + task id
- 状态用彩色标签（confirmed=绿、pending=黄、failed=红）
- 工作表数用图标+数字
- 结构版本用小字标注

### 2.2 编辑工具区域
**现状：** 按钮堆叠无分组，用户困惑用途。
**目标：**
- 按钮分三组：
  1. 区域操作（合并选中区域、拆分选中区域）
  2. 标记操作（标记为表头、标记为数据）
  3. 提交操作（清除选择、保存草稿、确认结构）用分隔线隔开
- 每个按钮加上简短说明
- 确认结构提示用醒目样式

### 2.3 结构预览区域
**现状：** 表头和数据单元格无颜色区分。
**目标：**
- 表头行用 dimension 背景色
- 数据区用 measure 浅色背景
- 合并单元格用边框加粗
- 选中区域高亮

## 三、去 AI 味样式修复（P1）

**用户明确要求：** 暖色手工纸质感，禁用 AI 味圆角渐变。

**需修改：**
| 问题 | 现状 | 目标 |
|------|------|------|
| 圆角过大 | border-radius: 28px | 改为 2px（直角） |
| 渐变按钮 | linear-gradient(...) | 改为纯色 var(--accent) |
| 卡片阴影过大 | box-shadow: 0 22px 60px | 改为 0 2px 8px |
| 玻璃态模糊 | backdrop-filter: blur(10px) | 移除 |
| 装饰性光晕 | 伪元素径向渐变圆 | 移除或大幅缩小 |
| 字体大小过大 | clamp(38px, 6vw, 84px) | 改为 clamp(28px, 4vw, 48px) |

## 五、完善上传后全流程交互（P1）

**每个阶段要有清晰的 UI 状态：**

1. 上传阶段（UploadPage）
   - idle -> 拖拽区
   - uploading -> 进度条 + 百分比
   - success -> 成功提示 + 文件元信息 + 进入结构确认按钮
   - error -> 失败提示 + 重试按钮

2. 结构确认阶段（TaskReviewPage）
   - loading -> 骨架屏/加载中
   - loaded -> 工作表列表 + 网格预览 + 编辑工具
   - saving -> 按钮 loading 态
   - confirmed -> 管线自动开始执行

3. 管线执行阶段（PipelinePanel）
   - 7 步逐步执行，每步显示 pending -> running -> done/failed
   - 全部完成后显示查看结果按钮

4. 结果展示阶段（ResultsPage）
   - loading -> 骨架卡片
   - loaded -> 概览卡片 + 图表 + 问题列表 + 原表联动
   - error -> 错误信息 + 返回按钮
