## 目标

补齐前端上传入口，让用户可以通过浏览器上传 .xlsx 文件并进入后续流程。

三种核心交互：
1. **拖拽上传** — 拖拽 .xlsx 文件到区域或点击选择
2. **进度条** — XHR upload.progress 实时百分比反馈
3. **结果提示** — 成功显示 Task ID + 跳转按钮，失败显示错误 + 重试

---

## 技术方案

### 1. useFileUpload hook
- XHR 上传（支持 progress 事件）
- 状态机：idle → uploading → success | error
- 返回 `{ uploadState, progress, result, errorMessage, upload, reset }`

### 2. UploadPage 组件
- 拖拽区域：dragenter/dragleave/dragover/drop 事件 + dropCounter 防闪烁
- 上传成功后点击"进入结构确认"→ 自动调 POST /{id}/parse → 再跳转 review
- 解析中显示动画进度条，失败显示错误

### 3. App.tsx 改造
- 新增 `upload` 阶段作为首页
- 上传成功自动跳转 review 阶段
