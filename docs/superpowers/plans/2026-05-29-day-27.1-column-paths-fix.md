## 背景

汇总服务读取 structure version snapshot 的 sheets 数据时，
column_kinds 和 column_paths 始终为空 []，导致 summary_builder 的 build_statistical_summary() 循环不执行，
columns 返回 []。下游 AI 分析无素材、图表推荐无数据 → 结果页全空。

## 根因分析

### 数据流链路

1. GET /review → build_sheet_payload() → with_header_parsing() ✅ 有 column_kinds/column_paths
2. 前端接收 sheets → TaskReviewPage 展示编辑
3. 保存草稿 → buildStructureVersionSaveRequest() 构造请求体
   → 仅传 10 个字段，缺失 column_kinds、column_paths、dimension_columns、measure_columns、header_row_span
4. POST /structure-versions → save_structure_version() → snapshot_json={"sheets": request_sheets}
   → 快照中无 column_kinds/column_paths ❌
5. POST /summarize → summarize_task() → structure.snapshot_json["sheets"]
   → sheet_data.get("column_kinds", []) → [] ❌
6. build_statistical_summary(..., [], [], ...) → columns=[] ❌
7. AI 分析、图表推荐 → 无数据 ❌
8. 结果页 → 空 ❌

### 责任位置

| 位置 | 文件 | 问题 |
|------|------|------|
| 前端 | frontend/src/modules/review-grid/structureVersionPayload.ts | buildStructureVersionSaveRequest 未传 column_kinds/column_paths |
| 后端 | backend/app/services/summarize/summarize_service.py | 直接从 snapshot 读，无兜底推导 |
| 后端 | backend/app/services/structure_version_service.py | save_structure_version 不校验/补齐字段 |

## 第二轮修复 — 数据驱动列类型推断

### 根因深化

虽然第一轮修复让 column_kinds/column_paths 能正确从 snapshot 推导，
但 Task #18 的 ligned_cell_roles 全为 dimension（结构确认时未正确标记数值列）。

uild_column_paths() 完全依赖 ligned_cell_roles 判断列类型，
导致所有列被标记为 dimension，uild_statistical_summary() 不计算任何统计指标。

### 修复

**文件**: ackend/app/services/grid_builder.py

新增 _infer_kind_from_data() 函数：
- 检查 header 行以下的每个单元格
- 若 >= 60% 的非空值匹配数字格式 → 推断为 measure
- 在 uild_column_paths() 中，若 
ole == "dimension" 但数据推断为 measure，覆盖
