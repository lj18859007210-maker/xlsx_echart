# 需求冻结清单

## 文档目标

冻结第一版产品边界，明确“先做什么、不做什么”，避免项目初始化后频繁返工。

## 用户角色

- 业务分析人员：上传表格、查看异常和分析结论
- 财务或经营管理人员：核对校验问题、关注图表和摘要结果
- 内部运营或实施人员：在 Gate 1 阶段修正表格结构，帮助系统进入后续自动分析流程

## 输入格式

第一版输入范围如下：

- 必须支持 `.xlsx`
- 默认输入为规则化 Excel 表格
- 允许存在多 sheet
- 允许存在合并单元格
- 暂不要求支持 OCR 图片表格识别

## 输出物

第一版必须输出以下结果：

- 清洗后的表格视图
- 结构化网格预览结果
- 校验错误清单
- 异常清单
- 分析结论
- 图表推荐结果
- 图表渲染结果

## 处理链路

统一任务主线为：

`upload -> parse -> confirm -> infer-formula -> validate -> detect-anomaly -> summarize -> analyze -> chart`

链路说明如下：

1. `upload`：上传文件并初始化任务。
2. `parse`：读取 workbook、sheet、单元格值、合并区域和基础结构信息。
3. `confirm`：进入 Gate 1，由用户进行结构确认和必要纠错。
4. `infer-formula`：根据表头结构和样本数据生成候选公式规则。
5. `validate`：由 Python 确定性执行校验逻辑。
6. `detect-anomaly`：执行规则型或统计型异常检测。
7. `summarize`：压缩结果上下文，形成摘要和异常切片。
8. `analyze`：调用 AI 生成可解释的业务分析结论。
9. `chart`：生成图表推荐和图表配置结果。

## 第一版范围

第一版明确包含：

- Excel 文件上传与解析
- 表格结构化预览
- Gate 1 人工结构确认
- 基础公式推导与校验
- 异常检测
- AI 分析结论
- 图表推荐与渲染

第一版默认按以下场景设计：

- 通用经营分析表
- 财务类规则化表格

## 非目标

第一版明确不做：

- OCR 图片识别主流程
- 多租户
- 复杂权限体系
- 移动端
- 批量异步大规模任务编排
- 可视化规则编排器
- 行业深规则一次性全覆盖

## 异常处理原则

- 上传失败时，必须返回明确错误原因。
- 解析失败时，必须指明是文件问题、结构问题还是读取问题。
- 结构不确定时，必须停在 Gate 1，不能直接跳过确认。
- 公式推导失败时，允许降级为空规则集，但必须保留失败记录。
- 图表生成失败时，允许只输出文本分析结果，但不能静默失败。

## 验收标准

- 用户上传一个标准 Excel 后，系统能输出结构化结果、问题列表、分析文本和至少 1 张合理图表。
- 合并单元格不会导致后续求和和聚合结果翻倍。
- 用户可以在 Gate 1 页面修正结构并继续流程。
- 校验结果可以追溯到具体规则和具体单元格。
- 异常结果具备可读原因，而不是只有分数或标签。
- 图表具备明确数据来源和解释，不为出图而出图。
- 任一阶段失败时，系统都能返回明确错误说明和当前最小可用结果。

## 技术栈确认

- 后端：`FastAPI`
- 数据处理：`Pandas`、`OpenPyXL`
- 数据模型：`Pydantic`
- 前端：`React`
- 图表：`ECharts`
- 数据库：`SQLite`
- 缓存：`Redis`（可选）
- AI：`LLM API`
