# 用户主流程

## 流程目标

定义第一版用户从上传 Excel 到查看分析结果的完整主流程，作为产品、前端、后端后续设计和联调的统一依据。

## 主流程

1. 用户进入系统并上传一个 `.xlsx` 文件。
2. 系统创建任务，保存文件信息并返回任务标识。
3. 系统解析 workbook、sheet、单元格、合并区域和基础样式。
4. 系统生成初始结构化网格预览，并进入 `waiting_confirm`。
5. 用户在 Gate 1 页面查看原表和结构化网格。
6. 如果结构有误，用户执行微调、合并、拆分、区块标记等修正操作。
7. 用户确认结构后，系统进入后续自动分析链路。
8. 系统推导候选公式并执行确定性校验。
9. 系统执行异常检测、摘要压缩和 AI 分析。
10. 系统输出问题清单、分析结论和图表结果。

## 用户视角的关键节点

### 1. 上传

用户只需要提供标准 Excel 文件，不需要先做复杂预处理。

### 2. 结构确认

这是第一版最关键的人机协作节点。系统不能假设自己一次性识别正确，必须允许用户在进入后续分析前纠正结构。

### 3. 结果查看

用户最终看到的结果应至少包括：

- 清洗后的表格视图
- 校验错误清单
- 异常清单
- AI 分析结论
- 图表推荐与渲染结果

## 主流程对应的系统动作

| 用户动作 | 系统动作 | 目标 |
| --- | --- | --- |
| 上传文件 | 创建任务并保存文件 | 建立可追踪任务 |
| 查看预览 | 返回原表和结构化网格 | 让用户发现结构问题 |
| 修正结构 | 生成新的结构版本 | 保证结构可回滚 |
| 确认结构 | 放行后续自动分析 | 降低级联错误 |
| 查看结果 | 返回问题、结论和图表 | 提供可解释输出 |

## 成功标准

- 用户可以独立完成上传并看到结构化预览。
- 用户可以在 1 到 3 分钟内完成单张表的结构修正。
- 用户可以从结果页追溯到异常来源和对应表格位置。
- 用户可以理解图表为什么被推荐，而不是只看到图形本身。
