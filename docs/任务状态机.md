# 任务状态机

## 文档目标

定义第一版任务在后端处理过程中的状态集合、状态迁移和失败原则，作为接口设计和任务编排的统一依据。

## 状态定义

### `uploaded`

文件已上传，任务已创建，但尚未完成结构解析。

### `parsed`

文件解析完成，系统已获得 workbook、sheet 和网格基础信息。

### `waiting_confirm`

系统已生成结构化预览，等待用户在 Gate 1 确认或修正结构。

### `confirmed`

用户已确认结构，允许进入公式推导、校验和后续分析阶段。

### `validated`

公式校验和异常检测已完成，系统已生成结构化问题结果。

### `analyzed`

摘要压缩和 AI 分析已完成，系统已生成可展示的结论结果。

### `chart_ready`

图表推荐和图表配置已完成，任务已具备完整展示条件。

### `failed`

任务在任意阶段发生不可继续错误，必须记录失败原因。

## 状态迁移

标准迁移路径如下：

`uploaded -> parsed -> waiting_confirm -> confirmed -> validated -> analyzed -> chart_ready`

任一阶段如果出现不可恢复错误，都可以迁移到：

`failed`

## 状态迁移说明

### `uploaded -> parsed`

触发条件：

- 文件格式通过基础校验
- 文件可被正常读取

### `parsed -> waiting_confirm`

触发条件：

- 已成功生成原表数据和结构化网格预览
- 已具备 Gate 1 展示所需数据

### `waiting_confirm -> confirmed`

触发条件：

- 用户完成结构修正或确认无需修正
- 当前结构版本被标记为确认版本

### `confirmed -> validated`

触发条件：

- 候选公式推导完成或降级为空规则集
- 校验和异常检测执行完成

### `validated -> analyzed`

触发条件：

- 已生成可供 AI 使用的摘要上下文
- 已输出基础问题清单

### `analyzed -> chart_ready`

触发条件：

- 图表推荐完成
- 图表配置可被前端直接消费或转换渲染

## 失败原则

- 上传失败：记录文件格式、大小、存储或请求错误。
- 解析失败：记录读取失败原因和失败 sheet。
- 结构失败：如果无法可靠识别结构，必须停留在 `waiting_confirm`，而不是跳过确认。
- 校验失败：如果规则无法执行，必须输出明确失败原因。
- 分析失败：如果 AI 分析失败，应尽量保留 `validated` 阶段产物。
- 图表失败：允许没有图表，但要说明失败原因，并保留文本结果。

## 状态机设计原则

- 状态必须可追踪，不能出现隐式跳转。
- 状态必须可恢复，尤其是 `waiting_confirm` 阶段。
- 状态必须服务于可解释性，而不是只服务于内部流水线。
- 状态名应同时让产品、后端、前端都能理解。
