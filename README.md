# xlsx_echart

一个面向 Excel 表格分析的系统，用于完成表格接入、结构识别、人工确认、公式校验、异常检测、AI 分析和图表生成。

## 产品目标

第一版目标是构建一条稳定、可解释、可追踪的表格分析主链路，让用户上传 `.xlsx` 文件后，可以得到：

- 清洗后的表格视图
- 校验错误清单
- 异常清单
- 分析结论
- 图表推荐与渲染结果

系统采用“LLM 负责理解与解释，Python 负责确定性计算，前端承接人工确认”的混合架构，优先保证结果可靠性，而不是一开始追求覆盖所有复杂场景。

## 第一版范围

- 输入：优先支持 `.xlsx`
- 表格类型：优先支持通用经营分析表、财务类规则化表格
- 核心流程：上传 -> 解析 -> 结构确认 -> 公式推导 -> 校验 -> 异常检测 -> 摘要 -> 分析 -> 图表
- 输出：结构化结果、问题清单、分析结论、图表结果

## 非目标

第一版暂不包含以下内容：

- OCR 图片表格识别
- 多租户
- 复杂权限体系
- 移动端适配
- 大规模批量异步任务编排
- 可视化规则编排器

## 用户主流程

1. 用户上传 Excel 文件。
2. 系统解析 workbook、sheet、单元格和合并区域。
3. 系统生成结构化网格预览，进入 Gate 1 人工确认。
4. 用户修正结构并确认后，系统继续执行公式推导和规则校验。
5. 系统输出校验问题、异常问题、摘要结果、AI 分析结论和图表结果。

## 任务状态

第一版统一任务状态如下：

- `uploaded`
- `parsed`
- `waiting_confirm`
- `confirmed`
- `validated`
- `analyzed`
- `chart_ready`
- `failed`

## 技术栈

- 后端：`FastAPI`
- 数据处理：`Pandas`、`OpenPyXL`
- 数据模型：`Pydantic`
- 前端：`React`
- 图表：`ECharts`
- 存储：`SQLite`
- 缓存：`Redis`（可选）
- AI 能力：`LLM API`
